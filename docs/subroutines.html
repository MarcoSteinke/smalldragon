SmallDragon Subroutine Declarations

SmallDragon uses 'fn' to declare and define a subroutine/function.

Example 


fn main () ~> Int {
	return 0;
}

In this Example, 'main' is the name of the subroutine, it takes no arguments, and the return Type is 'Int'. The subroutine is impure, as indicated by '~>', which means it is allowed to have observable side effects.

Example 2 


fn add (Int a, Int b) -> Int {
	return a+b;
}

In this case, 'add' takes 2 arguments, and produces no side effects.

-help

Nothing to say here

-avr

Uses 'avr-gcc' and 'avr-objcopy' to generate a 'main.hex' file which can be flashed onto some Microcontrollers or debugged/executed by AVR Simulators and Debuggers like mdx.

-debug

Prints Debug Statements during the execution of 'sd'. This flag is also propagated to 'dragon-lexer' and 'dragon-parser' which are called using 'system(...)' by 'sd'.

-clean

Re-generates all Artifacts used for incremental compilation, e.g. '.X.tokens', '.X.ast' .
This is useful if you changed the transpiler / gcc flags or are developing SmallDragon. Or, if you are paranoid of incremental compilation.

-dumpdot, -astdot

These flags are not yet implemented. They are intended to produce .dot files of the call graph and the AST respectively, so they can be easily inspected with the graphviz package.

-stdout

Prints the transpiled C Code to stdout, instead of writing it to a file.
This flag is incompatible with -debug.
Usage Example: "sd -stdout Test.dg | astyle"

-version

Prints SmallDragon version aswell as gcc version used to build it and the date on which it was built.
